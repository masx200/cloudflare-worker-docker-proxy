## 使用说明

### 1. Docker Hub 代理（`******************************`）

用于拉取 Docker Hub 上的镜像：

```bash
# 拉取官方镜像（如 nginx）
docker pull ******************************/library/nginx:latest

# 拉取其他镜像
docker pull ******************************/library/redis:latest
docker pull ******************************/library/mysql:8.0
```

**支持的镜像格式**：

- `library/镜像名:标签` — 官方镜像
- `用户名/镜像名:标签` — 用户镜像

---

### 2. GHCR 代理（`******************************`）

用于拉取 GitHub Container Registry 上的镜像：

```bash
# 拉取 GHCR 镜像（如 home-assistant）
docker pull ******************************/home-assistant/home-assistant:latest

# 其他示例
docker pull ******************************/owner/repo:tag
```

---

### 3. 配置 Docker 使用代理（可选）

如果你想让所有 `docker pull` 命令自动走代理，可以配置 Docker daemon：

**Linux** — 编辑 `/etc/docker/daemon.json`：

```json
{
  "registry-mirrors": [
    "https://******************************"
  ]
}
```

然后重启 Docker：

```bash
sudo systemctl restart docker
```

**注意**：`registry-mirrors` 只能配置一个默认镜像源。如果需要同时拉取 Docker Hub
和 GHCR 的镜像，建议直接在使用时指定完整域名（如上面的示例）。

---

### 4. 验证代理是否正常工作

```bash
# 测试 Docker Hub 代理
curl https://******************************/

# 测试 GHCR 代理
curl https://******************************/
```

正常应返回 `Docker Proxy is Running`。
